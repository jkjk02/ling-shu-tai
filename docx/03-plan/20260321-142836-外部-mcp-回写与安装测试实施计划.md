## 实施步骤

1. 扩展 `DiscoveryService`：
   - 判定 discovered MCP 是否可写。
   - 支持从候选根目录创建 flat JSON MCP 文件。
   - 支持更新与删除已发现的 flat JSON MCP 文件。
2. 修改 `mcps.py`：
   - `POST` 支持 `source_kind=discovered` 的外部创建。
   - `PUT` / `DELETE` 对可写 discovered MCP 走外部文件逻辑。
   - 保持 managed 路径不变。
3. 补充测试：
   - 成功路径：创建外部 MCP、更新 discovered MCP、删除 discovered MCP。
   - 失败路径：不可写 discovered MCP 更新被拒绝。
4. 前端与文档：
   - 调整 `MCPs` 页面描述，不再把 discovered 固定描述为只读。
   - 保持现有按钮条件，仅依赖后端返回的 `is_writable`。
   - README 补充“支持 flat JSON 外部 MCP 回写”的说明。

## 验证计划

1. `backend/.venv/bin/python -m unittest discover -s backend/tests`
2. `npm run build`
3. 如环境允许，再执行 `npm run verify`

## 完成定义

1. 现有 discovered MCP fixture 返回 `is_writable=true`。
2. 自动化测试证明外部 MCP 可创建、更新、删除且不污染 `backend/data`。
3. 前端构建通过。
