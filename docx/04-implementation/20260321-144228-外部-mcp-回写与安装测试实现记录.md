## 目标

1. 补齐受支持外部 MCP 的创建、回写、删除闭环。
2. 用隔离 fixture 覆盖可写成功路径和只读拒绝路径。
3. 保持 managed MCP CRUD、discovered 发现链路与前端构建稳定。

## 核心内容

1. 后端实现：
   - `backend/app/services/discovery.py`
     - 新增外部 MCP 创建、更新、删除能力。
     - 写能力收紧为候选根目录下 `mcps/` 的 JSON 文件。
     - 对 discovered MCP 保留 `_raw_payload`，回写时尽量沿用原字段别名。
   - `backend/app/routers/mcps.py`
     - `POST /mcps` 支持 `source_kind=discovered` 的外部 MCP 创建。
     - `PUT` / `DELETE` 对可写 discovered MCP 走外部文件逻辑，对只读 discovered 返回结构化 `mcp_read_only`。
2. 测试与样本：
   - `backend/tests/support.py`
     - 把 fixture 复制到临时目录后再启动真实 `uvicorn`，避免污染仓库样本。
   - `backend/tests/test_api_regression.py`
     - 覆盖 discovered MCP 回写、外部创建、重复创建冲突、外部删除和只读 `403` 分支。
   - `backend/tests/fixtures/cludea/models/locked.json`
     - 新增只读负样本，用于验证“发现成功但不在写支持矩阵内”的拒绝路径。
3. 前端与说明：
   - `src/pages/McpsPage.vue`
     - 页面描述更新为“受支持的外部 JSON MCP 可直接回写”。
     - 新建表单新增“写入目标”切换，支持 `managed` / `external`。
   - `README.md`
     - 更新自动化回归和当前实现范围，明确支持受限范围内的外部 MCP 回写。
4. 验证结果：
   - `npm run verify` 通过。
   - backend tests `6/6` 通过。
   - `npm run build` 通过。
