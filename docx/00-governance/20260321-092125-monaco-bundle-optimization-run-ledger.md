## 目标

1. 为“Monaco 包体优化”切片建立独立运行台账。
2. 在不回退 Monaco 体验的前提下，削减当前前端产物中的无效 Monaco worker 负担。
3. 固定本轮范围、验证门槛和交付链。

## 核心内容

1. 当前项目状态：
   - Monaco 已接入三处编辑位并通过构建。
   - 构建产物中出现体积巨大的 `ts/css/html` worker，但当前页面并未使用对应语言能力。
2. 本轮优先级判断：
   - 这是一个低风险、高收益的后续优化。
   - 无需改后端和业务表单逻辑。
3. 本轮范围：
   - 收缩 Monaco worker 与语言导入到当前真实用到的集合
   - 重新构建并记录优化结果
4. 本轮非范围：
   - 不做懒加载重构
   - 不改变 Monaco 接入页面
   - 不重写 Vite chunk 策略

## 风险与待确认项

1. 过度裁剪可能误伤未来需要的语言支持，因此必须只按当前已接入场景收缩。
2. 构建警告可能仍存在，但只要关键体积显著下降即可接受。

## 交接输出

1. 允许需求阶段围绕“只保留当前使用中的 Monaco 能力”建立需求基线。
2. 要求架构阶段明确当前真实使用语言集合，并比较“只裁 worker”与“进一步做懒加载”的取舍。
