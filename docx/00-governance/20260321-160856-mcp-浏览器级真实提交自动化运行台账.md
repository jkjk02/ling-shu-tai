## 目标

1. 为“managed MCP 浏览器级真实提交自动化”建立独立切片台账。
2. 在既有浏览器冒烟与后端回归已完成的基线上，补齐 MCP 页面真实提交链路。
3. 保持本轮只做最小可验证切片，不顺带推进 Workflow 拖拽或真实远端 CI。

## 核心内容

1. 当前产品状态：
   - 外部 MCP 回写已完成。
   - 浏览器级页面冒烟已完成。
   - `npm run verify` 已通过并接入浏览器测试。
2. 本轮切片范围：
   - 覆盖 `MCPs` 页面 `managed` MCP 的真实创建提交。
   - 覆盖创建后列表刷新与详情区校验。
   - 覆盖删除清理，确保重复执行不残留测试数据。
   - 浏览器测试后端运行时改为隔离数据目录，避免污染默认 `backend/data`。
3. 本轮非范围：
   - 不覆盖 discovered/external MCP 浏览器级写回。
   - 不覆盖 MCP 编辑链路。
   - 不覆盖 Workflow 画布拖拽/保存自动化。

## 风险与待确认项

1. 浏览器测试需要本地回环网络权限，沙箱环境下需提权执行。
2. MCP 页面表单来自 Element Plus，若选择器过宽，Playwright 严格模式会导致假失败。
3. 当前仅覆盖成功路径，不覆盖冲突、只读拒绝和异常恢复矩阵。

## 交接输出

1. 需求阶段必须锁定为 `managed MCP` 真实提交最小闭环。
2. 架构阶段必须明确采用“隔离数据目录 + 真实浏览器提交 + 删除双保险”方案。
3. 计划阶段必须把验证要求锁定到 `npm run test:browser` 和 `npm run verify`。
