## 操作说明书

### 启动

1. 前端开发：`npm run dev`
2. 后端开发：`backend/.venv/bin/python -m uvicorn app.main:app --app-dir backend --reload`

### 配置

1. 初始化前端依赖：`npm ci`
2. 初始化后端虚拟环境：`python3 -m venv backend/.venv`
3. 安装后端依赖：`backend/.venv/bin/pip install -r backend/requirements.txt`
4. 若需要触发远端 CI，将当前改动推送到带有 `.github/workflows/ci.yml` 的 GitHub 分支，或在 GitHub Actions 中使用 `workflow_dispatch`

### 常见操作

1. 本地统一验证：`npm run verify`
2. 本地 CI 仿真：`CI=1 npm run verify`
3. 查看浏览器报告：打开 `playwright-report/index.html`
4. 查看 JUnit 结果：检查 `test-results/playwright/results.xml`

### 验证方式

1. 本轮已验证：`CI=1 npm run verify`
2. 首次真实远端验证：推送分支或手动触发 GitHub Actions 后，检查 workflow run 状态与 artifact

### 已知限制

1. 本会话没有直接替用户触发 GitHub 远端 run，因此首次远端执行仍需要用户自行推送或手动触发。
2. 当前 CI 仍是单 job，不包含多平台、多版本或多浏览器矩阵。
3. 当前 artifact 成功与失败都会上传，后续如需控制存储成本可继续收敛。
4. 更复杂外部 MCP 方言适配仍未覆盖。

## 风险与待确认项

1. GitHub hosted runner 的后续镜像变化仍可能带来小幅适配成本。
2. 真实远端首轮运行仍建议观察一次 artifact、安装时长和浏览器稳定性。
